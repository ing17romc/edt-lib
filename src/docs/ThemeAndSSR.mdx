# Theme and SSR

`ThemeProvider` is the runtime theming boundary for the library.

Use it when you need:

- `light`, `dark`, or `system` theme mode
- a stable SSR fallback theme
- a consistent theme context across multiple components

Recommended setup:

```tsx
import 'edt-lib/styles.css';
import { ThemeProvider } from 'edt-lib';

export function AppProviders({ children }: { children: React.ReactNode }) {
  return (
    <ThemeProvider defaultTheme="system" ssrFallbackTheme="light">
      {children}
    </ThemeProvider>
  );
}
```

Guidance:

- prefer `styles.css` in Next.js and App Router
- keep the provider high in the tree
- avoid reading browser theme APIs during render from consumer code
